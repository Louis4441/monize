# v1.11.3

This release is mostly about the **AI Assistant**: you can now **attach files** — images, PDFs, and CSVs — to your questions, pop the chat open as a **floating bubble** on any page, and follow a confirmation card's **"View" link straight to the record it just touched**. Under the surface, the shared AI/MCP toolset got **another round of consolidation** so both surfaces stay in lockstep, and unknown account or category names now come back with **"did you mean?" hints**. It also lets **scheduled transfers carry a category**, adds a **custom-country picker** for manual ETF/fund allocation, makes **institution required** on loans and mortgages, and fixes a Monthly Breakdown column inconsistency.

## AI Assistant

### Attach images, PDFs, and CSVs to your questions

The AI chat is now **multimodal**: attach **images (PNG, JPEG, GIF, WebP), PDFs, and text files (CSV, plain text)** to a question by clicking the attach button, dragging a file in, or pasting one. Monize validates each file by size and type, decodes CSV/text inline, and maps the rest to each provider's native format (Anthropic, OpenAI, Ollama). Limits are **5 MB per file, 20 MB and 5 attachments per query**. If a model can't accept an image or a document, you get a **clear, localized message** instead of a silent failure. Attachments also work in **bring-your-own-subscription relay mode** — and PDFs there are **extracted to plain text server-side**, so the connected client (e.g. Claude Code) doesn't fall back to a local PDF handler that asks you to install extra tooling.

### A floating chat bubble on every page

A new **opt-in floating AI chat bubble** can follow you across the whole app. Enable it in **Settings → AI Settings** and a launcher button appears in the corner of every page (except the dedicated AI page), opening as a resizable bottom sheet that can expand to full screen. It shares the same conversation as the `/ai` page, is **lazy-loaded** so it stays out of the global bundle until you open it, and collapses automatically as you navigate. On desktop the chat sheet now **stays open across page navigation**. (Thanks **@WMP** for related fixes.)

### "View" links that land on the right record

After you approve a write, the confirmation card's **"View …" link now deep-links to the record it created or edited and briefly flashes it**, instead of dumping you on an unfiltered list. Transactions open on the page that contains the row; **payees, securities, and categories** flash in place via a shared highlight hook. Delete actions still have no link, and investments are deferred (their result is a transaction id, which the holdings list has no row for).

## Connect Monize to AI tools (MCP)

### A smaller, more consistent shared toolset

Another pass of consolidation on the **shared AI/MCP tool surface**, which must stay in lockstep:

- **Holdings folded into the portfolio summary** — the standalone holding-details tool is gone; `get_portfolio_summary` now returns a per-account breakdown, so one call answers both portfolio-wide and per-account questions.
- **Net-worth history, anomalies, and monthly comparison folded into `generate_report`** as report types, trimming the menu to **19 AI / 21 MCP tools**.
- **MCP investment read tools now accept account names** (resolved internally) instead of UUIDs, matching the AI Assistant and `list_transactions`.

### "Did you mean?" for accounts and categories

When an **account or category name doesn't resolve**, both surfaces now return a self-correcting **"did you mean?"** hint instead of silently dropping the filter (which would quietly scope the answer to the wrong records). The four `manage_*` output schemas were also factored into a shared builder (~110 fewer lines, no behavior change).

## Scheduled Transactions

### Scheduled transfers can carry a category

Following one-off transfers in v1.11.2, a **scheduled (recurring) transfer can now carry an optional category**, so a recurring contribution modelled as a transfer (e.g. a monthly investment top-up) shows up in the Monthly Breakdown without counting as income or expense. The category is kept on create, update, and posting — including the auto-post cron — and is stored on both legs.

### A tidier scheduled-transaction form

The scheduled-transaction form's **Active/Auto-post toggles and Cancel/Submit buttons now share one footer** across every tab, the **Investment tab fields were reflowed** into a more logical order, and the **Transfer tab gained an End Condition section** to match the Transaction tab.

## Securities & Portfolio

### Type your own countries for ETF/fund allocation

The manual ETF/fund **geographic allocation picker** now offers a **curated country list plus any custom countries you've already saved**, with your base-currency country floated to the top, and lets you **type a custom country name** directly into a new row. Freshly typed values are committed even when you click straight onto the submit button.

## Accounts

### Institution required for loans and mortgages

Creating a **loan or mortgage account now requires an institution**, with clean, localized validation messages instead of raw form errors. The backend resolves institution names from the Institution table while staying backward-compatible with legacy free-text values. (Closes #785.)

## Bug Fixes

- **Monthly Breakdown** showed different figures for the same month depending on the range preset, because day-level presets resolve to a mid-month start; the report now **snaps the start down to the first of the month**, so every column is fully covered and consistent across presets.
- A typed **payee or category is now kept** when you tab or click out of the combobox, instead of being discarded.
- **Backslashes are escaped before pipes** in markdown table cells, so the AI's tables render unambiguously.
- Added an **i18n fallback** for unknown insight types and fixed an `act()` warning in the geographic-allocation tests.

## Under the Hood

- **Security advisory**: bumped TypeORM to patched **0.3.30** (GHSA-9ggv-8w38-r7pm).
- New `pdf-parse` dependency backs server-side PDF text extraction for relay attachments.
- Database migration `088_user_preferences_ai_bubble_enabled.sql` adds the chat-bubble opt-in preference.
- Expanded test coverage across attachment validation and provider mapping, the chat bubble, the consolidated report types, name-resolution hints, scheduled-transfer categories, and the country picker, with i18n parity kept across all 21 locales.
